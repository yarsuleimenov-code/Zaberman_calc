Vehicle Types

Vehicles available for automatic vehicle fit and operational planning.

Назначение: справочник доступных vans/trucks для automatic vehicle fit.

Pricing engine использует Capacity, Max Weight и Fuel Type для подбора транспорта и расчёта operational cost.

Active checkbox отключает транспорт для новых расчётов без удаления исторических данных.

Packaging Types

Packaging options available for item-level quote creation.

Назначение: справочник типов упаковки, доступных брокеру при создании quote.

Packaging Type выбирается на item-level. Стоимость упаковки и коэффициенты должны применяться через pricing engine.

Custom Crate должен быть связан с отдельной crating formula: material + labor.

Item Catalog

Common item templates with predefined dimensions and handling defaults.

Назначение: справочник типовых предметов для ускорения работы брокера.

Item template может подставлять default dimensions, weight, packaging и handling flags. Брокер должен иметь возможность изменить значения вручную в quote.

Это особенно полезно для частых items: sofa, dining table, monitor, cabinet, mirror.

Service Areas

Zone-level pricing values used by route and regional cost calculations.

Назначение: справочник service areas с числовыми regional cost values.

Используются конкретные значения по зоне. Ппозволяет управлять tolls, parking и ticket risk отдельно для каждого региона.

Если fee не применяется, значение должно быть 0.

Insurance Types

Insurance options shown to broker during quote creation.

Назначение: справочник доступных insurance options.

Insurance Type выбирается в quote. Расчёт fee выполняется pricing engine через variables.

Basic Liability покрывает $0.60/lb, Full Coverage зависит от declared value.

Цель страницы

References.html является будущей admin-панелью управления справочниками. Она определяет, какие значения доступны брокеру и pricing engine при создании quote.

Разделение References и Variables

References отвечают за доступные значения: vehicle types, packaging types, item templates, zones и insurance types. Variables отвечают за ставки, коэффициенты и формулы расчёта.

Active / Inactive Logic

Active checkbox позволяет отключать значение без физического удаления. Отключённое значение не должно использоваться в новых расчётах, но должно оставаться доступным в historical estimates.

Regional Rules

Service Areas используют numeric regional values Это позволяет задавать tolls, parking, ticket risk и surcharge отдельно для каждой зоны.

Если regional component не применяется, значение должно быть 0. Это упрощает pricing formula и снижает количество IF-условий.

Best Practice

В production справочники должны храниться в database reference tables, использоваться через IDs и управляться admin role. Pricing engine должен работать с IDs и active status, а не с текстовыми значениями.
