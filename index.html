Назначение страницы

Основной рабочий экран брокера для подготовки customer estimate.

Основной workflow: route → items → access conditions → estimate generation.

Основные принципы

  • route считается по ZIP codes;
  • items являются главным pricing input;
  • services считаются отдельно для каждого item;
  • crew size рассчитывается автоматически;
  • pricing engine использует effective volume.

Item-level logic

Packaging, storage, insurance и crate requirement работают отдельно для каждого item.

Это позволяет корректно считать mixed orders.

Crew calculation

Crew зависит от веса, access conditions и handling complexity.

Additional helpers считаются минимум на 2 часа.

Access conditions

Stairs, elevator access и long carry влияют на labor cost, crew size, handling surcharge и final operational cost.

eBOL integration

Item list используется как база для будущего eBOL workflow.

Photos и comments помогают pickup/delivery verification.

Breakdown logic

Breakdown page показывает объяснение того же estimate, без отдельного пересчёта другой цены.
